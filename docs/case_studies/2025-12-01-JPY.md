# 2025-12-01 JPY Stress Window Case Study

## Executive Summary

This case study is the current strongest ANAS v1 demonstration.

FrictionGraph identified a clear JPY-heavy FX triangle stress window around
`2025-11-28` and `2025-12-01`. ANAS then searched license-safe official events
around that window and found that Japan/JPY/rates-related official events ranked
at the top of the alignment table.

The strongest focal event is:

```text
2025-12-01
Bank of Japan Governor Ueda speech
"Japan's Economy and Monetary Policy"
```

This event aligns well with the `2025-12-01` FrictionGraph response because:

- the event is directly about Japan's economy and monetary policy
- ANAS assigns high exposure to `monetary_policy`, `macro_growth`, `inflation`, and `JPY`
- FrictionGraph's top chains on `2025-12-01` are dominated by JPY-containing triangle chains
- the event-aligned chain is the second-largest RMSD chain of the day

Important boundary:

```text
This is an explanatory alignment case.
It does not claim that the BOJ speech caused the JPY move.
It does not claim to be a trading signal.
```

## Alignment Result

For the BOJ Governor Ueda speech:

```text
event_id: boj_2025_12_01_ueda_nagoya_speech
event_date: 2025-12-01
selected_window: same_day
friction_date: 2025-12-01
daily_sum_rmsd: 0.004020865281
top_rmsd_chain: EUR -> SGD -> JPY -> EUR
top_alignment_chain: USD -> SGD -> JPY -> USD
top_alignment_chain_rmsd_rank: 2
top_alignment_currency_overlap: 160.0
top_alignment_weighted_currency_overlap: 8700.0
alignment_score: 2.9543254593
```

This is strong because the event-aligned chain is not a weak chain that merely
contains JPY. It is the second-largest RMSD chain on the stress date.

The alignment formula used in this first demo is:

```text
weighted_currency_overlap =
  sum(currency_score[ccy] * currency_impact[ccy]
      for ccy in chain_currencies)

alignment_score =
  weighted_currency_overlap * chain_rmsd
```

For `USD -> SGD -> JPY -> USD`:

```text
chain currencies: USD, SGD, JPY

USD contribution from event struct:
  score 60 * impact 40 = 2400

SGD contribution:
  score 10 * impact 0 = 0

JPY contribution:
  score 90 * impact 70 = 6300

weighted_currency_overlap:
  2400 + 0 + 6300 = 8700

chain_rmsd:
  0.000339577639

alignment_score:
  8700 * 0.000339577639 = 2.9543254593
```

This provides a transparent bridge from official-event text to FrictionGraph
chain behavior.

## Why The Alignment Is Good

The alignment is good for four reasons.

### 1. The FrictionGraph Signal Is Clear

The stress window is not a small or ambiguous fluctuation. It is visually and
numerically prominent in the daily `sum_rmsd` series:

```text
2025-11-28 sum_rmsd = 0.003754249909
2025-12-01 sum_rmsd = 0.004020865281
```

The chain-level decomposition shows that JPY is not incidental. On `2025-12-01`,
the top 10 chain-level RMSD rows all contain JPY.

### 2. The Official Event Is Directly Relevant

The primary event is not a generic macro release. It is a BOJ Governor speech
on Japan's economy and monetary policy on the same date as the larger
FrictionGraph spike.

That gives ANAS a clean official-source event anchor:

```text
official text -> monetary policy / Japan macro exposure -> JPY exposure
```

### 3. The Event-Aligned Chain Has High Market Rank

For the Ueda speech, the top alignment chain is:

```text
USD -> SGD -> JPY -> USD
```

Its RMSD rank on `2025-12-01` is:

```text
rank 2 of 32 chains
```

That matters. ANAS is not finding alignment in a low-ranked residual chain.
It is finding alignment in one of the largest market-structure responses of the
day.

### 4. The Control Logic Works

The top case-study ranking is dominated by Japan/JPY/rates events:

```text
1. BOJ Noguchi speech, 7-day window, score 4.0749
2. MOF JGB issuance revision, next-day window, score 3.9059
3. BOJ Ueda speech, same-day window, score 2.9543
```

Later or more indirect events, such as the `2025-12-10` Fed FOMC statement and
BOJ CGPI release, score lower in this JPY stress window case.

This supports the ANAS design claim:

```text
JPY/Japan/rates official events rank higher against a JPY-heavy FrictionGraph
stress window than less directly related events.
```

## Important Edge Case: MOF FX Intervention Disclosure

The MOF FX intervention disclosure on `2025-11-28` is highly relevant but has
alignment score `0` in the current scoring method.

Reason:

```text
currency_score["JPY"] = 100
currency_impact["JPY"] = 0
```

The official disclosure states that foreign exchange intervention operations
for the period were `¥0`. The LLM therefore treated the event as highly
relevant to JPY/FX, but neutral in impact.

This is useful because it reveals a design improvement:

```text
ANAS should separate relevance / attention from directional impact.
```

Future struct:

```json
{
  "currency_relevance": {
    "JPY": 100,
    "USD": 20
  },
  "currency_impact": {
    "JPY": 0,
    "USD": 0
  },
  "interpretation": "Highly relevant FX disclosure, but neutral result because intervention amount was zero."
}
```

This would allow the system to say:

```text
The event is very relevant to JPY market monitoring, but it should not be
treated as an active intervention shock.
```

That is the correct behavior for a professional demo.

## Portfolio Interpretation

This case demonstrates the value of combining ANAS and FrictionGraph:

```text
FrictionGraph detects where FX market structure became abnormal.
ANAS identifies which official events have exposure vectors that line up with
that abnormal chain structure.
```

For `2025-12-01`, the key result is:

```text
BOJ monetary-policy communication
  -> high JPY and Japan macro exposure
  -> alignment with top-ranked JPY-heavy FrictionGraph chains
```

The result is not a prediction engine yet. It is a structured, auditable
alignment layer that can support:

- case-study generation
- Tableau annotation
- event-window comparison
- future supervised fine-tuning or preference alignment data
- portfolio demonstration of source policy, LLM extraction, market-structure analytics, and explainable ranking

## Recommended Next Improvement

The next structural improvement should be:

```text
Split current score/impact logic into:
  relevance_score
  directional_impact_score
  market_attention_score
```

This would make the MOF FX intervention disclosure behave more naturally:

- high relevance
- neutral impact
- still important for explanation

That improvement will make ANAS stronger as both a research system and a
portfolio project.
